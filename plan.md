# Logit 開発計画書

## プロジェクト概要

**Logit（ロジット）** は、AI時代における「思考のIDE（統合開発環境）」です。
Gitの仕組みを人間の思考プロセスに適用し、AIに頼り切らず「考えること」を楽しみ、思考力を鍛えるプラットフォームを目指します。

---

## 1. コアバリュー・差別化

### コンセプト
```
既存ツール: 「何を考えたか」を記録するノート
Logit: 「どう考えたか」を鍛えるジム
```

### 4つの柱

#### ① 思考のバージョン管理システム
- **Git for Thought**: コードの変更履歴を管理するGitの概念を、思考プロセスに適用
- **論理的根拠の追跡**: 「なぜこの結論に至ったか」を時系列で記録
- **コミット**: 思考の断片、迷い、仮説を含めてスナップショット化
- **ブランチ**: 「もしこう考えたら？」という仮説思考を並行して育てる
- **マージ**: 異なる思考を統合し、自分なりの結論を導く

#### ② メタ認知能力の育成
- **思考の観察**: 「自分がどう考えているか」を俯瞰しながら考える習慣
- **過去との対話**: 「あのとき自分はこう考えていたのか」という振り返り
- **思考パターンの認識**: 自分の思考の癖や傾向を可視化
- **成長の実感**: 思考の進化を時系列で確認できる

#### ③ 思考の効率化（ループ・漏れの防止）
**問題1: 同じ思考のループ**
- 重要な決断を何度も考え直し、堂々巡りしてしまう
- 過去に検討した論点を忘れて、また最初から考え始める

**Logitの解決策**:
- 過去のコミット履歴から「この論点は3ヶ月前に検討済み」と通知
- 思考の重複を検知し、効率的な意思決定をサポート

**問題2: 考慮漏れ**
- いつも同じ観点を見落としてしまう
- 多角的な検討ができない

**Logitの解決策**:
- AI Challengeが「この観点は考えた？」と指摘
- ブランチ機能で複数の仮説を並行検討
- 過去の履歴から「前回はこの論点を見落としていた」と学習

**結果**: ロジカルシンキング力の向上、考える力の筋トレ

#### ④ AIとの協働による思考深化

**AIの位置づけ**:
```
❌ AI = 答えを出す存在（従来）
✅ AI = 思考を揺さぶる触媒（Logit）
```

**3つのAI視点**:
- **Boost（ポジティブ）**: 可能性を広げる質問、関連する成功事例の提示
- **Challenge（ネガティブ）**: 論理的矛盾の指摘、反証、リスク分析
- **Synthesize（中立）**: 客観的データ・統計、過去の類似思考との比較

**重要**: 最終的な統合・判断はユーザー自身が行う

### 既存ツールとの差別化

| ツール | 焦点 | Logitとの違い |
|--------|------|--------------|
| Obsidian/Roam | 知識のデータベース | 結果は蓄積されるが、プロセスは見えない |
| Notion | 情報の整理棚 | 完成形を重視、試行錯誤は残らない |
| Scrapbox | アイデアのネットワーク | 繋がりは見えるが、時系列や分岐は追いづらい |
| ChatGPT/Claude | 答えの自販機 | 即答は得られるが、思考筋力が育たない |
| **Logit** | **思考の成長記録 + トレーニング環境** | **プロセスを資産化、メタ認知を育成** |

### 具体的な使用例

**シナリオ: 「転職すべきか」を3ヶ月考えている人**

**従来のアプローチ**:
- 毎回ゼロから「メリット・デメリット」を考え直す
- 堂々巡りして決断できない
- 過去に何を考えたか思い出せない

**Logitでのアプローチ**:
```
1月: 「年収アップがメリット」と考える
     → Challenge AI: 「でも業界の将来性は？」
     → ブランチ分岐: 業界研究を開始

2月: 「やっぱりリスクが...」
     → Logitが通知: 「この論点は1月に検討済みです」
     → 過去のコミットを確認: 「ああ、リスク軽減策も考えてたんだ」

3月: 新しい視点「ワークライフバランス」を追加
     → 過去の論点と統合（マージ）
     → 総合的な判断基準が確立
     → 納得して決断
```

### 提供価値まとめ

- **論理的思考力の向上**: 根拠を明確にする習慣
- **メタ認知能力の育成**: 自分の思考を俯瞰する力
- **意思決定の質向上**: 考慮漏れやループを防ぐ
- **思考の資産化**: 過去の思考プロセスを再利用可能に
- **AI時代の思考筋トレ**: 答えに頼らず、自分で考える力を維持・向上

## 2. ターゲットユーザー

### 基本方針

**使用スタイル**: 日記型ではなく、**ツール型**
```
❌ 毎日書く習慣（Notion、日記アプリ）
✅ 必要なときに使う道具（Git、IDE）
```

**使用パターン**:
- **集中モード**: 30分〜2時間、じっくり考える時間を取る
  - 例: 週末に重要な決断を考え抜く

- **隙間時間モード**: 5〜15分、思考の断片を記録
  - 例: 通勤中にふと浮かんだアイデアをコミット
  - 後でまとめてマージ

- **振り返りモード**: 過去の思考履歴を確認
  - 例: 「前回このテーマを考えたときはどうだったか？」

### 優先ターゲット層

#### ① ソフトウェアエンジニア（最優先）

**なぜ相性が良いか**:
- **Gitの概念に既に慣れている** → 学習コストが低い
- コミット、ブランチ、マージが直感的に理解できる
- 技術的な意思決定が日常的にある
- ドキュメンテーション文化がある

**具体的な使用シーン**:
```
例1: 技術選定
「ReactとVueどっちを使うか？」
→ ブランチで両方の選択肢を検討
→ Challenge AI: 「チームの習熟度は考慮した？」
→ Synthesize AI: 「過去のプロジェクトでは...」
→ 統合して判断、その根拠を記録

例2: アーキテクチャ設計
「マイクロサービスにすべきか、モノリスか？」
→ 思考のコミット履歴を残す
→ 半年後の振り返り: 「あのときこう考えて決めたんだ」
→ 意思決定の質が向上

例3: リファクタリング判断
「この技術的負債、今直すべき？」
→ 過去に同じ議論をした履歴を確認
→ 「前回は後回しにして失敗したな」
→ 学習して判断
```

**ペインポイント**:
- 「なぜこの設計にしたか」を後から思い出せない
- 技術選定の根拠が曖昧で、レビューで説明できない
- 同じような悩みを何度も繰り返している
- ChatGPTの一般的な回答では、プロジェクト固有の判断ができない

---

#### ② 知的生産者層

**対象**: 研究者、ライター、コンサルタント、企画職

**使用シーン**:
- 論文の論理構成を練る
- 記事のアイデアを深掘りする
- 企画提案の根拠を固める
- クライアント向け提案の思考整理

**ペインポイント**:
- ChatGPTに頼ると自分の思考が薄まる
- 論理の穴に気づけない
- アイデアが散らかって整理できない
- 「なぜこの結論に至ったか」を説明する必要がある

**Logitで解決**:
- 思考のブランチで複数の仮説を並行検討
- 論理の矛盾をAI Challengeが指摘
- 最終的な統合（マージ）で、納得のいく論理構成を構築
- 思考プロセスそのものがドキュメントになる

---

#### ③ 自己成長志向層

**対象**: 学生、若手社会人、スキルアップ志向の人

**使用シーン**:
- キャリア選択の思考整理
- 学んだことの深い理解
- 問題解決能力の訓練
- ロジカルシンキングの実践

**ペインポイント**:
- AIに頼りすぎて思考力が低下している不安
- ロジカルシンキングを実践したいが場がない
- 自分の成長を実感できない
- 「考える力」を具体的にどう鍛えれば良いか分からない

**Logitで解決**:
- 思考のプロセスを可視化して、メタ認知能力を育成
- ゲーミフィケーションで「考えること」を楽しむ
- 過去の思考と比較して、成長を実感
- AI時代における「思考筋トレ」の場

---

### ペルソナ例: ソフトウェアエンジニア

**名前**: 田中太郎（28歳）
**職業**: Webエンジニア（経験5年）
**所属**: 中規模スタートアップでフロントエンド担当

**状況**:
- 技術選定やアーキテクチャ判断を任されることが増えた
- チームメンバーからの質問に答える立場になってきた
- 「なぜこの技術を選んだか」を後から説明する必要がある

**現在の課題**:
- 技術選定の根拠が感覚的で、言語化できない
- 過去に同じような判断をしたはずだが、思い出せない
- ChatGPTに聞くと一般的な答えしか返ってこない
- Notionにメモしているが、思考のプロセスは残らない

**Logitを使った変化**:
1. 技術選定のたびに思考プロセスを記録
2. AI Challengeで「この観点は考えた？」と指摘される
3. 半年後、同じような判断が必要になったとき、過去の履歴を参照
4. 「前回はこう考えて、結果はこうだった」という学習サイクル
5. チームメンバーへの説明が論理的になり、信頼が向上

**典型的な1週間の使い方**:
- 月曜: 新機能の技術選定について30分考える（集中モード）
- 火曜: 通勤中にふと浮かんだ懸念点をコミット（隙間時間モード）
- 水曜: AI Challengeのレビューを読んで、新しいブランチで代替案を検討
- 金曜: 各ブランチをマージして最終判断、チームに共有
- 日曜: 過去の技術選定履歴を振り返り、パターンを発見（振り返りモード）

## 3. 成長の可視化（ゲーミフィケーション）

### 基本方針

**コンセプト**: ゲームではなく、**「思考のGitHub」**

GitHubがエンジニアのコード活動を可視化するように、Logitは思考活動を可視化する。
過度な演出はせず、シンプルで実用的に、エンジニアに馴染みのあるUIで成長を実感できるようにする。

### 6つの可視化機能

#### ① 思考アクティビティ（GitHub草風）

```
[ビジュアルイメージ]
2025年の思考活動
Jan  ■□■■■□□ ■■■□■■□
Feb  ■■□■□■■ □■■■■□■
Mar  ■■■■■■■ ■■■■■■■
     少←──────────→多
```

**機能**:
- 日々の思考活動（コミット、ブランチ、マージ）を可視化
- 色の濃さで活動量を表現
- クリックすると「この日は技術選定について3時間考えた」と詳細表示
- ストリーク表示: 「7日連続で思考活動中！」

**効果**:
- 習慣化の促進（「今日も何か考えよう」）
- 振り返りやすい（「先月はどんなことを考えていたか」）
- シンプルだが効果的

---

#### ② 思考スキルプロフィール

```
[レーダーチャート]
       論理的思考
            ★
           /|\
          / | \
多角的 ★─────★ 深堀り
視点    \  |  /  思考
         \ | /
          \|/
           ★
      メタ認知
```

**測定するスキル**:
- **論理的思考**: 根拠の明確さ、論理の一貫性
- **多角的視点**: ブランチ数、異なる視点の検討数
- **深堀り思考**: コミットの深さ、Challenge AIへの対応率
- **メタ認知**: 過去の思考との比較頻度、パターン認識

**表示方法**:
- 初期値: すべて低い（成長の余地を見せる）
- 思考活動に応じて自動的に上昇
- 「3ヶ月前と比較」ボタンで成長を可視化
- 「あなたの強み: 論理的思考、弱み: 多角的視点」と表示

**効果**:
- 自分の思考特性を客観的に把握
- 弱点を意識して強化できる
- 長期的な成長を実感

## 4. UX・体験設計

### 基本方針

**開発優先順位**:
1. **Web UI優先**: まずはブラウザベースのアプリケーションを開発
2. **CLI対応も念頭**: 将来的にCLI操作できるような設計（API設計を意識）
3. **Gitライクなデータ構造**: CLI移行時にスムーズに対応できる

**参考モデル**:
- Git (CLI) + GitHub (Web UI)
- Obsidian (ローカル) + Obsidian Publish (Web)

---

### AIレビューのタイミング（コスト削減）

```
❌ リアルタイムでAIレビュー（高コスト）
✅ PR（レビュー依頼）時のみAI呼び出し

Git風のフロー:
1. 思考をコミット（複数回、AIは呼ばない）
2. ある程度まとまったら「レビュー依頼」（PR作成相当）
3. このタイミングでAIが3視点レビュー（コスト発生）
4. レビューを見て、さらに思考を深める
5. 必要に応じて再度レビュー依頼
```

**効果**:
- AI API呼び出しを最小化
- ユーザーが明示的にレビューを依頼する仕組み
- 考えがまとまってからレビューを受ける方が効果的


**実装**:
- テキスト入力中、リアルタイムでVector Search
- Embedding APIでベクトル化（OpenAI, Claude等）
- Vector DB（Cloudflare Vectorize等）で検索
- 類似度の高い過去の思考を表示
- **LLMは使わないので低コスト**

**データフロー**:
```
1. ユーザーが思考をコミット
   ↓
2. テキストをEmbedding APIでベクトル化
   ↓
3. Cloudflare Vectorizeに保存
   ↓
4. 新しい思考を書くとき、リアルタイムでベクトル検索
   ↓
5. 類似度の高い過去の思考を表示
```

**コスト試算**:
- Embedding API: 非常に安い（LLMの1/100以下）
- Vector Search: ほぼ無料（Cloudflare Vectorizeの無料枠）
- リアルタイム提案でも、月数ドル程度

## 6. 思考の資産化（検索・再利用）

### 基本方針

**全文検索ではなく、構造化された検索とAI支援**

従来の全文検索の問題:
- ヒット数が多すぎる
- 意味的な関連性が分からない
- 探している思考が見つからない

Logitのアプローチ:
- トピック/タグベースの整理
- Vector Searchによる意味的検索
- AI自動提案

### データモデル（Phase 1）

#### テーブル構成

```sql
-- ユーザー（Phase 1では簡易実装）
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  username TEXT UNIQUE NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 思考トピック
CREATE TABLE topics (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  title TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- ブランチ
CREATE TABLE branches (
  id TEXT PRIMARY KEY,
  topic_id TEXT NOT NULL,
  name TEXT NOT NULL,
  parent_branch_id TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (topic_id) REFERENCES topics(id),
  FOREIGN KEY (parent_branch_id) REFERENCES branches(id),
  UNIQUE(topic_id, name)
);

-- コミット
CREATE TABLE commits (
  id TEXT PRIMARY KEY,
  branch_id TEXT NOT NULL,
  content TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (branch_id) REFERENCES branches(id)
);

-- AIレビュー
CREATE TABLE reviews (
  id TEXT PRIMARY KEY,
  branch_id TEXT NOT NULL,
  boost TEXT NOT NULL,
  challenge TEXT NOT NULL,
  synthesize TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (branch_id) REFERENCES branches(id)
);
```
